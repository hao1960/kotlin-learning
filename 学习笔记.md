# Kotlin 学习笔记

## 已学内容

### Day 1 — 基础语法
- `val` / `var`
- 基本类型、`fun`、`main`
- `data class`
- `List` / `mutableListOf`、遍历与手动搜索

### Day 2 — 集合操作
- `filter`：筛选
- `map`：转换
- `find`：找第一个，没有则 `null`（`Song?`）
- `sortedBy`：排序

### Day 3 — 面向对象
- `interface`：只定义能力，方便面向接口编程
- `abstract class`：可带公共实现（如 `current`）
- 扩展函数：`fun Song.displayName()`（无 `extension` 关键字）
- 字符串插值：`$name` 与 `${song.name}`

### Day 4 — Lambda
- 函数类型：`(Song) -> Unit` / `(Song) -> Boolean`
- 高阶函数：把函数当参数传递
- 尾随 Lambda：`foo(list) { ... }`
- 单参数可用 `it`

### Day 5 — 协程
- `suspend`：可挂起（里面能 `delay` / 模拟网络）
- `launch`：启动协程，一般不要返回值
- `async` + `await`：要结果；多个 async 可并行等待
- `Flow` + `collect`：陆续接收多个异步数据
- `runBlocking`：练习用，让 `main` 等协程跑完

## 注意事项（复习易错点）

1. **有返回值的函数记得 `return`**  
   建了 `result` 却忘了 `return result`（Day1 / Day4 都出现过）。

2. **`mutableListOf` 要带类型**  
   `mutableListOf<Song>()` / `mutableListOf<String>()`，不要空泛写成 `mutableListOf()`。

3. **扩展函数不要写 `extension`**  
   正确：`fun Song.displayName(): String = ...`

4. **Lambda 不要多包一层**  
   错：`filterSongs(songs) { song -> { song.singer == "周杰伦" } }`  
   对：`filterSongs(songs) { song -> song.singer == "周杰伦" }`  
   括号外的 `{ }` 本身就是 Lambda。

5. **运行前先保存文件（Cmd+S）**  
   未保存时 `gradle run` 跑的是磁盘旧代码。

6. **按天运行**  
   ```bash
   gradle run -Pday=05
   ```

7. **协程打印顺序可能交错**  
   `launch` 与 `async` 并发时，谁先打印完取决于各自耗时，属正常。

8. **`launch` 不要拼成 `lauch`**
